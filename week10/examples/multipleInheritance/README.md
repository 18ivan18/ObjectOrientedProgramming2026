# Многократно наследяване и Diamond проблемът

## Многократно наследяване

C++ позволява един клас да наследява от **повече от един базов клас** едновременно. Това е полезно, когато даден клас трябва да комбинира поведението на няколко независими йерархии.

```c++
class Flier  : public Animal { /* може да лети */ };
class Swimmer : public Animal { /* може да плува */ };

class Duck : public Flier, public Swimmer { /* и двете */ };
```

## Diamond проблемът

Проблемът възниква, когато два базови класа споделят един и същ прародител:

```
       Animal
      /       \
  Flier      Swimmer
      \       /
        Duck
```

Без специална мярка компилаторът ще вгради **два отделни** `Animal` подобекта в `Duck` — по един от `Flier` и по един от `Swimmer`. Всяка от двете копия пази собствени стойности на `age` и `weight`:

```c++
Duck d("Ivan");
d.age;           // грешка — кое age? Flier::Animal::age или Swimmer::Animal::age?
d.Animal::age;   // пак неясно — кой Animal?
```

Освен неяснотата при достъп, имаме и излишно дублиране на данни и двойно извикване на конструктора/деструктора на `Animal`.

## Виртуално наследяване

Решението е да се декларира наследяването от общия прародител като `virtual`:

```c++
class Flier  : virtual public Animal { ... };
class Swimmer : virtual public Animal { ... };
```

При виртуално наследяване компилаторът гарантира, че в `Duck` ще има **точно един** `Animal` подобект, споделен от `Flier` и `Swimmer`. Конструкторът на `Animal` се извиква само веднъж — директно от `Duck`.

```c++
Duck d("Ivan");
d.age;    // OK — има само един Animal
```

### Важни правила

- Виртуалното наследяване се декларира в **класа наследник** (`Flier`, `Swimmer`), не в прародителя (`Animal`).
- При виртуално наследяване **най-отдалеченият наследник** (`Duck`) отговаря за инициализацията на виртуалния базов клас — дори да не го наследява директно.
- Виртуалното наследяване добавя малко допълнителен overhead (допълнителен указател за намиране на споделения подобект).

## Примерът в тази директория

Йерархията тук моделира патица, която може и да лети, и да плува:

```
Serializable  (интерфейс: serialize / deserialize)
     |
   Animal     (age, weight, virtual print())
   /    \
Flier  Swimmer   ← и двата наследяват virtual public Animal
   \    /
    Duck        ← наследява Flier и Swimmer; има само един Animal
```

`Duck` е единственият конкретен клас в йерархията и имплементира всички чисто виртуални методи: `print()`, `serialize()`, `deserialize()`. Благодарение на виртуалното наследяване `Duck` пази само по едно копие на `age` и `weight`.
